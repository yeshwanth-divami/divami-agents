# Output Format — Vyasa

- Callouts: `> [!note] Title`, `> [!warning] Title`, `> [!tip] Title` — Obsidian syntax only. `///` is not supported.
- Foldable detail: `<details><summary>Why X over Y</summary>...</details>`
- Cross-document links: relative `.md` paths
- TOC is generated by Vyasa automatically — do not emit one
- **Prose line breaks: never.** Each paragraph is one unbroken line in the source file. Do not wrap prose at 80 characters or any other column limit. Vyasa renders line breaks as spaces, so artificial wrapping produces ugly rendered output.
- Diagrams: Mermaid or D2 — use literal `<br/>` for line breaks inside nodes
- Glossary links: `[term](glossary#term)` — no `.md` extension. From a module subdirectory: `[term](../../glossary#term)`

Read the `vyasa` skill for full syntax if needed.
