# Mode: Seed

Input: a topic, component name, system description, or rough notes.

Output:
1. A context seed paragraph (3–5 sentences) — see `writing-patterns.md` § Context Seed
2. A document skeleton: section headings with one-sentence stubs describing what each section will *say*, not what it will *contain*
3. One placeholder diagram slot with a note on which diagram type fits (sequence, ER, C4 level)

The context seed must pass the cold-reader test: cover the document title and show the seed to someone unfamiliar with the project. Can they follow the document? If yes, the seed works.
